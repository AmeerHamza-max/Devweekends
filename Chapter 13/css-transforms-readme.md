# CSS Transforms — Complete Guide

## What is a CSS Transform?

A CSS transform **visually repositions, resizes, or distorts** an element **without affecting page layout**. Other elements don't move — the transformed element is simply drawn differently on screen.

```css
.box {
  transition: transform 0.4s ease; /* always pair with transition for smooth animation */
}
.box:hover {
  transform: scale(1.2);
}
```

## The Bug in Your Original Code

### ❌ What You Wrote (Wrong)

```css
.box:hover {
  transform: rotate(360deg);
  transform: scale(2);       /* overwrites rotate */
  transform: scale(0.5);     /* overwrites scale(2) */
  transform: skew(45deg);    /* overwrites scale(0.5) */
  transform: translate(500px);
  transform: translate(0px, 500px);
  transform: translate(0px, -500px);
  transform: translate(500px, -500px); /* only THIS one runs */
}
```

**Why it breaks:** CSS is a "last wins" system. When you write the same property multiple times, only the last declaration is applied. All the lines above are silently ignored except the very last one.

### ✅ The Fix (Correct)

```css
.box:hover {
  transform: rotate(360deg) scale(2) translate(50px, 0); /* combine in ONE line */
}
```

## All CSS Transform Functions

### 1. `rotate(angle)`

Spins the element around its **transform-origin** (center, by default).

| Value | Effect |
|-------|--------|
| `rotate(45deg)` | 45° clockwise |
| `rotate(-90deg)` | 90° counter-clockwise |
| `rotate(1turn)` | Full 360° spin |

```css
.box:hover {
  transform: rotate(45deg);
}
```

---

### 2. `scale(x)` / `scale(x, y)` / `scaleX(x)` / `scaleY(y)`

Resizes the element from its center.

| Value | Effect |
|-------|--------|
| `scale(1)` | No change (default) |
| `scale(2)` | Double size (both axes) |
| `scale(0.5)` | Half size |
| `scale(-1)` | Flip both axes (mirror) |
| `scaleX(2)` | Stretch horizontally only |
| `scaleY(0.5)` | Squish vertically only |
| `scale(2, 0.5)` | Double wide, half tall |

```css
/* Hover grow effect */
.card:hover {
  transform: scale(1.05);
}

/* Flip effect (mirror image) */
.icon {
  transform: scaleX(-1);
}
```

---

### 3. `translate(x, y)` / `translateX(x)` / `translateY(y)`

Moves the element from its current position. **Does not affect other elements.**

| Value | Effect |
|-------|--------|
| `translate(50px, 0)` | 50px to the right |
| `translate(0, -20px)` | 20px upward |
| `translateX(100%)` | Move right by its own width |
| `translateY(100%)` | Move down by its own height |

```css
/* Center an element absolutely (classic technique) */
.centered {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* Slide in from left */
@keyframes slideIn {
  from { transform: translateX(-100%); }
  to   { transform: translateX(0); }
}
```

---

### 4. `skew(x, y)` / `skewX(x)` / `skewY(y)`

Slants the element along an axis, creating a parallelogram effect.

| Value | Effect |
|-------|--------|
| `skewX(20deg)` | Slant horizontally |
| `skewY(10deg)` | Slant vertically |
| `skew(20deg, 10deg)` | Both axes |

```css
/* Diagonal card design */
.hero-banner::after {
  content: '';
  transform: skewY(-3deg);
}
```

---

### 5. `perspective(n)` + `rotateX/Y/Z`

Creates **3D depth**. Must be first in the transform chain.

```css
/* 3D card flip */
.card:hover {
  transform: perspective(600px) rotateY(30deg);
}

/* Fold-down effect */
.panel:hover {
  transform: perspective(800px) rotateX(20deg);
}
```

| Value | Effect |
|-------|--------|
| `perspective(200px)` | Very dramatic depth |
| `perspective(800px)` | Moderate depth |
| `perspective(2000px)` | Subtle depth |

---

### 6. `rotateX(angle)` / `rotateY(angle)` / `rotateZ(angle)`

3D rotations along each axis. `rotateZ()` is identical to `rotate()`.

```css
.box:hover {
  transform: perspective(600px) rotateY(180deg); /* card flip */
}
```

---

### 7. `matrix(a, b, c, d, e, f)`

A single function that encodes ALL 2D transforms mathematically. Rarely written by hand — usually generated by JS animation libraries.

```
matrix(scaleX, skewY, skewX, scaleY, translateX, translateY)
```

```css
/* Equivalent to rotate(45deg) scale(1.5) */
.box {
  transform: matrix(1.06, 1.06, -1.06, 1.06, 0, 0);
}
```

---

### 8. `translate3d(x, y, z)` / `scale3d(x, y, z)` / `rotate3d(x, y, z, angle)`

3D equivalents of the 2D functions. A common performance trick:

```css
/* GPU acceleration hack */
.animated {
  transform: translate3d(0, 0, 0); /* forces GPU compositing layer */
  will-change: transform;           /* tells browser to prepare */
}
```

## Combining Transforms

**Always combine in ONE `transform` property, space-separated.**

```css
/* Order matters! Transforms apply left → right */
.box:hover {
  transform: translateX(50px) rotate(45deg) scale(1.2);
}

/* Different order = different result */
.box2:hover {
  transform: rotate(45deg) translateX(50px); /* translates along rotated axis */
}
```

### Common Combinations

```css
/* Hover: lift + enlarge */
.card:hover {
  transform: translateY(-8px) scale(1.02);
}

/* Spin and grow */
.icon:hover {
  transform: rotate(360deg) scale(1.3);
}

/* 3D tilt on hover */
.tile:hover {
  transform: perspective(500px) rotateX(10deg) rotateY(-10deg);
}
```

## transform-origin

Controls the **pivot point** for rotation and scaling.

```css
.box {
  transform-origin: center;        /* default: 50% 50% */
  transform-origin: top left;      /* rotate from top-left corner */
  transform-origin: bottom center; /* scale from bottom */
  transform-origin: 20px 80px;     /* exact pixel coordinates */
}
```

### Examples

```css
/* Door hinge effect (rotates from left edge) */
.door {
  transform-origin: left center;
}
.door:hover {
  transform: perspective(600px) rotateY(-90deg);
}

/* Grow from bottom */
.bar {
  transform-origin: bottom;
}
.bar:hover {
  transform: scaleY(2);
}
```

```css
}
```

Better practice — only transition `transform` (faster):

```css
.box {
  transition: transform 0.4s ease;          /* smooth ease */
  transition: transform 0.3s ease-in-out;   /* ease both ends */
  transition: transform 0.5s cubic-bezier(0.34, 1.56, 0.64, 1); /* springy */
}
```

## Performance Notes

| Property | Performance | Why |
|----------|-------------|-----|
| `transform` | ✅ Excellent | GPU composited, no reflow |
| `top` / `left` | ❌ Slow | Triggers layout reflow |
| `width` / `height` | ❌ Slow | Triggers layout + repaint |
| `opacity` | ✅ Good | GPU layer when combined with transform |

**Always use `transform` to move/resize elements in animations, never `top`/`left`.**

```css
/* ❌ Slow — causes layout reflow */
.box:hover { left: 50px; }

/* ✅ Fast — GPU composited */
.box:hover { transform: translateX(50px); }
```

## Real-World Usage

| Transform | Real Usage |
|-----------|------------|
| `translate` | Dropdown menus sliding in, modal slide-up, tooltips |
| `scale` | Button hover effects, image zoom on hover, notification badges |
| `rotate` | Loading spinners, hamburger menu → X icon, chevron arrows |
| `skew` | Hero section diagonal cuts, stylized card designs |
| `perspective` + `rotateY` | Card flip (front/back), 3D product showcases |
| `translate3d(0,0,0)` | GPU acceleration for scroll animations |
| `translateX(-50%) translateY(-50%)` | Perfect centering with `position:absolute` |
